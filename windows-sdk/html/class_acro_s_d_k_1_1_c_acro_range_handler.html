Detailed Description

Abstract range handler class.

This class is used in inline mode checks for coloring errors in your document, replacing text etc. Derive your class from AcroSDK::CAcroRangeHandler and implement all abstract methods.

virtual void AcroSDK::CAcroRangeHandler::invalidateRangeInformation ( const AcroSDK::CAcroRangeKeySet &  keySet,
AcroSDK::CAcroRangeKeySet &  replacmentRanges,
AcroSDK::CWStrVec &  rangeContents 
) [inline, virtual]

Validate keySet against the keys that actually have been marked and memorized internally by the plugin.

Let keySet be the set of all keys that are to be replaced in the subsequent replacement operation.

If supportsComplexReplacement(keySet) returns true, the SDK would call invalidateRangeInformation.

On call of invalidateRangeInformation "replacementRanges" is a copy of keySet and rangeContents holds the corresponding content of the ranges identified by replacementRanges.

It is possible that the plugin does not have any knowledge about some of the keys that are contained in keySet since the SDK concatenates subsequent ranges in order to save marking operations.

invalidateRangeInformation returns the set of keys "replacementRanges" that is a modified version of "keySet" where "replacementRanges" contains only those keys that are actually still known within the plugin.

The return value "rangeContents" should contain the corresponding content of those ranges that are known to the plugin.

COLORREF  color 
) [pure virtual]

Change color of given range.

The color of the text marking is defined by the corresponding error / issue (Grammar, spelling, etc.). In addition the SDK maintains an "active flag", similar to a cursor position. When using the inline check modus, the user iterates from flag (marked text) to flag. The currently active flag will be marked in a different color (yellow by default). For changing the color of a range, the SDK will call this method.

[in]keyUnique key of the range

virtual bool AcroSDK::CAcroRangeHandler::supportsComplexReplacement ( const CAcroRangeKeySet &  keySet) [inline, virtual]

Indicate support for complex replacement.

The plugin should return true if it would support a complex replacement for the ranges specified by "keySet". If supportComplexReplacements returns true, the SDK would call "invalidateRangeInformation" first, in order to find out what the plugin actually knows about "keySet". Then, the SDK would call replaceAndRemoveRanges with a replacement vector instead of a single replacement string in order to better preserve the document structure.
