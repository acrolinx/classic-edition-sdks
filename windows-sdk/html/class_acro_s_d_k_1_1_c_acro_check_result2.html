Detailed Description

Collects result of a check operation.

The results of a check operation are returned by an object of CAcroCheckResult2. Its properties contain summary information of the check. Call CAcroCheckResult2::correctRanges(...) for correcting the text using the corrections dialog. Call CAcroCheckResult2::createRanges(...) for correcting the text in inline mode.

Correct issues found by last check, using the corrections dialog.

After running a check, this method launches the corrections dialog. When the user has done all changes in the dialog, the resultDoc contains all changes. The text selection callback can be used for scrolling the original document in the background to the related text passage that is currently visible in the corrections dialog. Use the error object for getting more detailed information when correctRanges returns false.

Correct issues found by last check, using the inline mode (range handler)

After running a check, this method lets the range handler mark all issues in the original document. Use the error object for getting more detailed information when createRanges returns false. When the method returns, all issues have been marked in the document. At this point, you are done with the "Check text" command, the application should return to the main message loop. From here on, forward any check-related button clicks from the client application to the SDK. When the user presses any buttons in the application, call the appropriate methods (onNextFlag ...)

Returns:
True if all ranges have been created. Returns false when the user canceled the operation or any failure occurred.
Parameters:
[out]rngHandlerReference to range handler implementation
[in]bIsRecheckSet to true when this was a "recheck selection"
[out]errorDetailed error message that can be displayed to the user
[in]lMaxNumberOfRangesOptional: maximum number of supported ranges (it limited in the client application)
