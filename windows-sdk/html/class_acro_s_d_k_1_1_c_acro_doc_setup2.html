virtual void AcroSDK::CAcroDocSetup2::defineFilter ( const wchar_t *  filterName,
const CWStrSet &  filterTokenSet 
) [virtual]

Define an optional filter for text extraction.

For some purposes the user needs to exclude parts of the original document from checking. For example, when writing a programming tutorial, users can exclude source code fragments from the check to avoid a large number of false alarms. Similar to the discussed segmentation configuration, the SDK options dialog provides a tab for filter configuration. Again, the tag or style names are used for filling in the filter tab. You can add multiple filters for font names, style names, etc. by invoking AcroSDK::CAcroDocSetup2::defineFilter(...). The user can choose one of the defined filters in the options dialog.

Use NULL or empty filter name when the filter name corresponds to the doc type info.

Parameters:
[in]filterNameapplication-specific filter name, the string will not be displayed
[in]filterTokenSetstring vector of element names that can be filtered, e.g. XML tags, font names...

virtual void AcroSDK::CAcroDocSetup2::defineSegmentation ( const CWStrSet &  segmentationTokenSet) [virtual]

Define available elements for segmentation configuration.

For handling structured content like XML correctly, the SDK uses break levels for determining how blocks of text are separated from each other. The break level defines how a text block is related to the following block. The text extraction algorithm has to "translate" the application-specific text segmentation (text flow information ) into the SDK text model. The Acrolinxs Server is not aware how the submitted text is structured in the originating document. Therefore it is important to choose appropriate break levels for the text blocks. Text content of multiple spreadsheet cells is linguistically not linked. Usually one continuous sentence is not distributed over multiple cells.

This method defines the list of elements that are available for the current doc type. The SDK will use this list on the segmentation tab for configuration of the break levels.

Parameters:
[in]segmentationTokenSetstring vector of element names / XML tags
virtual void AcroSDK::CAcroDocSetup2::enableCheckStatusInsert ( bool  enable) [virtual]

Support embedding statistics of the last check.

Setting this option to true enables the checkbox "Embed checking status" on the plug-in tab of the options dialog. The user can then toggle this feature on / off. Only if we have enabled it and the user has switched it on, we will get check status values after the check. In case of a batch check, this will ensure that onUpdateFile(...) of the batch check callback will be invoked after a file has been checked. For a regular check in inline mode / with corrections dialog, CAcroCheckResult2::getCheckStatus() returns the data that should be written into the document attributes. When the checkbox "Embed checking status" is checked, CAcroCheckResult2::getCheckStatus() will return a list of properties to be embedded. When the checkox "Embed checking status" is not checked, CAcroCheckResult2::getCheckStatus() will return an empty list.

Parameters:
[in]enableset to true when embedding check results is possible for this document

Get list of empty elements that should be extracted as info blocks.

Info blocks are used for telling the server that there are elements in the text that cannot be checked. The SDK sends a so-called empty element to the server. The linguistic rules on the server recognize empty elements as valid linguistic content.getFilterEmptyTokenSet() returns the list of tag names that should be extracted as info blocks (empty elements).

Returns:
String vector of all elements (XML tags, style names...) that should be extracted as info blocks.

virtual const CWStrSet& AcroSDK::CAcroDocSetup2::getSegmentationNoBreakTokenSet ( ) [virtual]

Get list of elements with break level "no break".

The break level defines how a text or info block is related to the next block. No break: There is nothing in between this block and the next block. When the text itself does not contain separating characters like a space, the text will be concatenated.

Returns:
String vector of all elements (XML tags) that should be extracted with break level "no break".
virtual const CWStrSet& AcroSDK::CAcroDocSetup2::getSegmentationParentheticTokenSet ( ) [virtual]

Get list of elements with break level "parenthetic element".

The break level defines how a text or info block is related to the next block. Parenthetic element: The text is something like a footnote. The text is linguistically not linked to the left-hand or right-hand side of the originating position in the original document. The text must be extracted separately from the regular text flow. Typically this text is added to the SDK document after the normal text flow. The text blocks of a parenthetic element should be terminated with a block of break level "sentence break" in order to avoid unindented concatenation of multiple parenthetic elements.

Returns:
String vector of all elements (XML tags) that should be extracted as parenthetic elements.

Get list of elements with break level "Sentence break".

The break level defines how a text or info block is related to the next block. Sentence break: The end of a block can be considered as the end of a sentence. Typical example: Items of a bulleted list or a paragraph tag.

Set document type information (required for xml files)

The filter and segmentation settings are stored and applied based on the document type (dtd). When working on an XML file, set all known doc type infos so that the SDK can store and load the settings accordingly. Supported doc type infos are DOC_TYPE_XML__SYSTEM_ID (file name of the dtd), DOC_TYPE_XML__PUBLIC_ID ("-//OASIS//DTD DITA Topic//EN"), DOC_TYPE_XML__DOCTYPE_NAME ("topic"), DOC_TYPE_XML__SCHEMA_ID and DOC_TYPE_XML__ROOT_NAME. For non-xml files usefull settings are DOC_TYPE__DOCTYPE_NAME (e.g. 'application/word') and DOC_TYPE__TEMPLATE_NAME (e..g. 'normal.dot')

Parameters:
[in]infoNameThe parameter name. DOC_TYPE_XML__SYSTEM_ID ...
[in]infoValueThe parameter value. "c:\\my.dtd"
