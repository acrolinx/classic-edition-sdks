Detailed Description

Context menu entry.

For some use cases, the markings should be displayed in the document (inline mode) but the common context menu should be suppressed. For this case it is possible to retrieve the content of the suppressed context menu.

Execute the menu entry action.

const wchar_t* AcroSDK::CContextMenuEntry::getName ( ) const

Get menu entry name (type)

The menu entry name is a magic string that will give a hint about the related action. [AddToDictionary], [ContributeTerm] will add terms to the dictionary / term bank. [Edit], [Ignore], [IgnoreAll], and [IgnoreWholeDoc] will remove the marking via range handler callback. [Label] is the description of the issue and will open the help page in the web browser. [Next] and [Prev] will move the next or previous flag. You should suppress those entries if you don't want a visible context menu. [BeginPopUp] and [EndPopUp] mark the beginning and end of a pop menu. [Replace] and [ReplaceWholeDoc] will replace text. The suggestion will be given as the entry string (see above). [StepThroughMode] will toggle the step through mode. You should suppress this entry if you don't want a visible context menu. [SetMenuPosition] will only be visible in step through mode and is used for changing the menu position. You should suppress this entry if you don't want a visible context menu.

Returns:
The menu name (magic string).
