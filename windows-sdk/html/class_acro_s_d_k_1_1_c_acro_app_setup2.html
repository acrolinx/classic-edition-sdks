virtual void AcroSDK::CAcroAppSetup2::defineCharacterSubstitution ( wchar_t  fromChar,
wchar_t  toChar,
TSubstitutionMode  substMode 
) [virtual]

Replace characters before / after check.

Sometimes applications use special characters, like non-breaking spaces, special dashes or spaces with a certain length. The linguistic rules on the server do not know all possible representations of such special characters. The SDK supports character replacements that are applied automatically before a check, after a check or in both directions. For example the string "35 mm" in your document is replaced by choosing a suggestion from the Acrolinx shortcut menu or corrections dialog. The new string can contain a FrameMaker non-breaking space . This character should be replaced with a regular non-breaking space before it is written back to the document.

Parameters:
[in]fromCharthe char code in your application
[in]toCharthe char code as used by the SDK
[in]substModethe direction (replace before / after check)

virtual void AcroSDK::CAcroAppSetup2::enabledFilterAndSegmentationColors ( bool  enabled) [virtual]

When enabled, the SDK can apply coloring to filtered text and text with certain break levels.

The SDK can apply coloring to filtered text and text with certain break levels. This is useful for the end user for analyzing the results of the current filter and segmentation configuration. When enabled, the plug-in tab of the options dialog allows to apply colors to all included, excluded, and sentence-break elements according to the current filter and segmentation settings. This option should be enabled if the plug-in supports the inline checking mode. For "corrections dialog only" plug-ins, this option is not useful. For the user, this feature is bound to the privilege "Edit filter and segmentation settings".

Parameters:
[in]enabledset to true for enabling this feature

virtual unsigned int AcroSDK::CAcroAppSetup2::getLimitForNumberOfRanges ( ) const [virtual]

Get the maximum supported number of ranges (colored ranges in the document)

The maximum supported number of ranges for a client can be restricted via server settings or local registry. This value is only for informational purposes. The client does not need to care about this restriction. By default, the number of ranges is unlimited. Only when there are issues concerning the count of displayable ranges, one should limit the number of supported ranges. In this case, set DWORD value "RangeNumberLimit" in registry key HKLM[Wow6432Node]<YOUR_PLUG_IN_SHORT_NAME> to the max number of supported ranges. This should be done by the client setup.

Returns:
The supported number of ranges. Returns 0 when there is no limit.

virtual void AcroSDK::CAcroAppSetup2::useOneSessionPerCheck ( bool  enabled) [virtual]

Define behavior of allocating check sessions.

For most use cases, the client allocates a session on the Acrolinx Server when the user opens the options dialog or checks text. By default, the session is released when the client application terminates. For some use cases, it is more appropriate that the session is released after each check operation. Example: the Acrolinx Desktop Checker allocates a session when a string from the clipboard is checked. Afterwards the session is released immediately. Otherwise the session would be kept active until Windows is terminated. This method should be called only when the default behavior is not appropriate.

Parameters:
[in]enabledtrue: the session is closed after each check; false: the session is closed when the client application terminates
