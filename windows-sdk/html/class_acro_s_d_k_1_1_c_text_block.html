Detailed Description

Extracted text content.

A text block can be compared with the “text run” concept of the rich text model. A sequence of such text blocks is collected in an <codeph>AcroSDK::CTextDoc</codeph> instance and represents the whole document respective the current text selection. Besides the text content, a text block supports some additional parameters. Every text block has to be initialized with an application-specific key. The key can be used for linking the extracted text block to its original location within your document. Therefore you should use unique keys for each block.

Member Enumeration Documentation

enum AcroSDK::CTextBlock::CorrelationType

Correlation of inserted text block to the blocks around.

For some corrections, the SDK needs to insert new blocks into the corrected document. The correlation tells if the new block has to use the formatting of the block to the left- (idx - 1) or right-hand side (idx + 1).

Enumerator:
CT_BEFORE 

[in] new block should use the formatting of the block on the left-hand side

CT_ORIGINAL 

[in] the text of the original block is replaced with new text, replace text and keep formatting

CT_AFTER 

[in] new block should use the formatting of the block on the right-hand side

CorrelationType AcroSDK::CTextBlock::getCorrelation ( ) const

Get correlation of text block to the blocks around.

For some corrections, the SDK needs to insert new blocks into the corrected document. The correlation tells if the new block has to use the formatting of the block to the left- (idx - 1) or right-hand side (idx + 1).

Returns:
CT_ORIGINAL when it is the original block with / without changes. Returns CT_BEFORE or CT_AFTER when it is a new block and the formatting should be applied from the block before or after this block.
